架构说明

项目由 Worker、管理存储和路由 Hub 三部分构成，职责清晰，适合直接在边缘运行。

数据与控制面

管理存储与后台登录

`AdminStore` 保存路由配置、统计信息和事件记录；后台登录密码通过 Cloudflare Workers Secret 设置，推荐变量名为 `ADMIN_PASSWORD`。

POST /api/login { "password": "你的 ADMIN_PASSWORD" } GET /api/state 返回路由列表、统计和事件
连接与消息面

WSS 接入与 peer 协调

每条路由使用 `/ws/:routeId/:clientToken` 作为入口。Workers 会解析 EasyTier 的 peer 头信息， 并在同一路由的连接之间进行消息转发与协调。

wss://your-domain/ws/route-id/client-token 由 RouteHub 负责该路由的连接管理

部署说明

推荐的发布方式是 GitHub Fork + Cloudflare Workers 网页部署。整个过程可以完全不依赖本地命令行。

Fork 仓库

先把项目复制到你的 GitHub 账号

打开仓库页面，点击 Fork。后续 Cloudflare 会直接连接你 fork 出来的仓库。

Cloudflare

在 Workers 网页上导入仓库并创建项目

进入 Cloudflare Dashboard 的 Workers 创建页面，选择从 GitHub 导入项目，然后授权并部署。

Secret

设置后台密码变量

在 Worker 的 Secrets / Variables 中添加 `ADMIN_PASSWORD`，这就是管理后台的登录密码。

完成

访问 `/panel` 并创建路由

部署完成后，打开 `/panel`，登录后即可配置路由并生成 EasyTier 的 WSS 连接命令。

后台与使用方式

后台不仅能管理路由，也能帮你直接生成客户端使用命令，减少配置出错的概率。

后台能力
  • ✓路由管理：创建、编辑、删除、启用或禁用单条路由。
  • ✓命令生成：直接输出 EasyTier 客户端可用命令。
  • ✓状态查看：查看连接数、累计连接和最近事件。
示例命令
sudo easytier-core -d \ --network-name my-network \ --network-secret my-secret \ -p 'wss://your-domain/ws/route-id/client-token' 后台生成的命令可以直接复制使用

常见问题

把一些最常见的部署与定位问题放在这里，方便快速确认。

这个项目是代理别人的 WSS 服务吗？

不是。它本身就是一个直接运行在 Cloudflare Workers 上的 EasyTier WSS 支持实现。

后台密码应该怎么设置？

通过 Cloudflare Workers 的 Secrets / Variables 设置 ADMIN_PASSWORD。代码也兼容旧变量 ADMIN_SECRET，但推荐统一使用前者。

为什么推荐 GitHub Fork + Cloudflare 网页部署？

这样最适合这个项目的发布方式：仓库里保持完整源码，部署时由 Cloudflare 直接读取 GitHub 仓库，不要求你在本地执行部署命令。

GitHub Pages 这个介绍页怎么启用？

在仓库的 Settings → Pages 中，将发布源设置为 Deploy from a branch，分支选 main，目录选 /docs。保存后，GitHub Pages 就会从这个目录发布本介绍页。
